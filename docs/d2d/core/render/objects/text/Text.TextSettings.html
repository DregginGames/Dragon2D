    Struct Text.TextSettings

    This struct holds the settings for structs. Why this is needed you would ask? There are two ways to update text object. After every setting changed, the text needs to be rebuilt. That takes time. See the functions why thats not an easy thing. Next would be: why dont flush() the text? Flushing the text after each change forces one to carry the whole flush function, along with the setters to the object hierarchy. Thats not nice. I decidet it would be much nicer to just carry the Setting struct - a thing that cant be broken that easyly and the setter then takes care of the updates.

    Fields

    Name Type Description
    color gl3n.linalg.Vector!(float,4) THe color of the text
    font string font resource name
    height float The size of the text
    linebreak bool If the text will break on linefeeds. If set to true, multiline text becomes possible
    lineOffset float Offset-factor between different lines. (This value is, lets say, random.)
    maxheight float The maximal height of a text - used when linebreak is true. if set (>=0) the textnoxes height wont exeed a specific value
    maxwidth float The maximal length of the text. if not set (<=0), the whole text will render in one line until its end, no matter how long it is
    overflow Text.OverflowBehaviour Specifes how text-overflows will be handled
    positioning Text.Positioning Positioning of the text.
    scroll float How far the text is scrolled - if overflow is scroll. Better use texts scrollUp and scrollDown functions since these dont regenerate the text.
    scrollUnit float How big a single scrolling unit is (basically how far one line is in world coordinates. yes. )
    size d2d.core.resources.font.Font.FontSize font size name
