    API documentation

    Module Description
    d2d.core.container.gamecontainer A pure container class wich is the top-node of the game-related classes
    d2d.core.container.root d2d.root holds the root element of the engine
    d2d.core.dbg.eventdebug d2d.core.debug.eventdebug holds helpers to debug the event system
    d2d.core.render.lowlevel.buffer Holds the abstraction layer for buffer objects
    d2d.core.render.lowlevel.gputexture Contains the abstraction for gls textures
    d2d.core.render.objects.quad Simple renderables for rendering quads on screen
    d2d.core.render.objects.renderable d2d.core.render.renderable holds the base classes for object rendering.
    d2d.core.render.objects.text Renders text on screen
    d2d.core.render.objects.view d2d.core.render.view holds the view classes that manage the rendering of the scene on screen.
    d2d.core.render.util.general d2d.core.render.util provides utulity functions for 2D and 3D rendering
    d2d.core.render.lowlevel Package file for all lowlevel stuff
    d2d.core.render.objects The package module that *should* contain all valid render objects
    d2d.core.render.renderer d2d.core.render.renderer holds the classes used to render the whole engine.
    d2d.core.render.util Package header that includes all render utility files
    d2d.core.resources.font Holds a font resource
    d2d.core.resources.glslprogram d2d.core.resources.glprogram holds the resource that represents a glprogram;
    d2d.core.resources.texture d2d.core.resources.texture holds the texture resource types
    d2d.core.services.scheduler d2d.core.services.scheduler manages the scheudling of function calles and other timed mechanisms
    d2d.core.base d2d.base holds the base interfaces needed by most of the classes.
    d2d.core.callback d2d.core.callback holds the types and few functions needed for Dragon2Ds callback system
    d2d.core.event d2d.core.event holds the basic event class and the container for native sdl events
    d2d.core.io d2d.core.io holds the classes that abstract the interaction with the user
    d2d.core.resource d2d.core.resource contains the bases for all resources used by d2d
    d2d.game.dbg.grid Holds the debug class for grid rendering
    d2d.game.simple.camera d2d.game.simple.camera holds the default camera class
    d2d.game.simple.sprite d2d.game.simple.sprite holds the sprite class: a simple class to display images on screen
    d2d.game.ui.cursor d2d.game.io.cursor holds the cursor-functions
    d2d.game.ui.text
    d2d.game.ui.uielement Abstract base for all ui elements.
    d2d.game.ui.uievent Holds all ui events or events related to ui
    d2d.game.entity d2d.game.entity holds the base class for all objects actually appearing ingame: entitys
    d2d.system.env d2d.system.env holds the enviourment of D2D - opengl, sdl, sound system and all this stuff get insitialized by the env class.
    d2d.util.fileio d2d.util.fileio provides structs and functions for efficient file resoure management
    d2d.util.logger d2d.logger holds the basic functions for logging. It supports several output methods.
    d2d.util.settings d2d.settings manages all settings - from command line arguments to the config files of the engine
    d2d.engine The d2d.engine module conains the absoute basesystem of the Draon2D engine.
    game Main entry point for a game. Here a game should define its "scripting". This includes game logic, ui handlers, ...
